## Bridge Scripts

- `fewshot_qm_xiepai_common.py`: shared paths, split specs, feature/stage helpers.
- `prepare_qm_xiepai_fewshot_splits.py`: source inventory and S0-S8 split generation.
- `check_fewshot_splits.py`: count, leakage, and cumulative-stage checks.
- `export_qm_xiepai_clip_features.py`: CLIP-B16 official projected residual feature export plus CHMM cache.
- `run_fewshot_ahl_stage.py`: stage root construction and AHL invocation.
- `evaluate_fewshot_stage_metrics.py`: calibration-only threshold selection and fixed-test metrics.
- `summarize_fewshot_curve.py`: S0/S2/S4/S6/S8 summary table and trend judgement.
- `run_ahl_no_cudnn.py`: runtime wrapper for AHL; disables cuDNN and supplies minimal optional dependency/runtime compatibility patches.

Bridge compatibility notes:

- S0 uses `--auxiliary False` because the original AHL auxiliary reward branch has no real anomaly samples in zero-anomaly training and produced NaN after epoch 7.
- Very small known-anomaly stages keep the requested anomaly count; the wrapper allows replacement only when AHL internally asks `random.sample` for more elements than exist in that temporary support/query subset.
- `--backbone resnet18` is passed only to satisfy AHL feature dimension lookup for cached 512-channel features; the extracted feature backbone remains CLIP-B16 official projected residual feature.
- `matplotlib` and `einops` stubs are installed only for missing optional imports in the remote conda/PBS runtime.

## Trend Judgement

- Precision did not improve monotonically as anomaly samples increased. It changed from 0.8667 at S0 to 0.7167 at S8, with a local recovery at S6.
- Recall improved materially from S0 to S8: 0.2167 -> 0.7167. The highest recall in the first round was S4 at 0.8667.
- F1 improved from S0 to S8: 0.3467 -> 0.7167, but peaked at S4 with 0.7879 and then dropped at S6/S8 under the calibration best-F1 threshold policy.
- Under a strict practical gate of Precision/Recall/F1 >= 0.80/0.80/0.80, no stage fully qualifies.
- For pilot triage, S2 and S4 are the first practically interesting stages:
  - S2 is balanced without anomaly calibration: P/R/F1 = 0.8462/0.7333/0.7857.
  - S4 has the strongest recall and best F1: P/R/F1 = 0.7222/0.8667/0.7879.
- Diminishing or unstable returns begin after S4: S2 -> S4 F1 gain is only 0.0022, and S4 -> S6 is negative.
- The full-supervised switch gate of Precision/Recall/F1 >= 0.90/0.90/0.90 was not reached.

## Final Answers

1. Precision 是否提升：没有稳定提升。S0 到 S8 从 0.8667 降到 0.7167，S6 有 0.8571 但 Recall 下降到 0.6000。
2. Recall 是否保持稳定或提升：总体提升，S0 到 S8 从 0.2167 到 0.7167；最高点在 S4，为 0.8667。
3. 哪个阶段开始具备实际可用性：严格按 P/R/F1 都不低于 0.80，第一轮未达到；若作为早期产线辅助筛查，S2/S4 开始具备试用价值，S4 更偏召回优先。
4. 哪个阶段后收益开始变小：S4 后收益开始变小或不稳定；S2 到 S4 的 F1 增益已经很小，S6/S8 回落。
5. 是否达到切换全监督模型条件：未达到。没有任何阶段达到 P/R/F1 >= 0.90/0.90/0.90。
6. CLIP-B16 + AHL + CHMM 是否适合作为产线早期小样本方案：适合作为冷启动到全监督前的过渡性筛查方案，但不适合作为当前阈值策略下的最终稳定生产模型。
7. 下一步：优先做 threshold calibration 和 full-supervised baseline；随后做 DRA+CLIP score fusion，对比是否能稳定提高 S4-S8 的 Precision/Recall trade-off。
