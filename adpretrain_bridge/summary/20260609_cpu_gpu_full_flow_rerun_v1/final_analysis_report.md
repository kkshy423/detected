## 2. 方法关系

- **ADP-only-DINO**: training-free 残差匹配基线。使用正常参考图建立 DINO/projector 残差参考库，对 query 做 patch 级最近邻残差评分。本项目固定 `num_ref=8`。
- **AHL-DINO**: 在 ADP/DINO 投影残差 feature cache 上训练 few-shot 检测器。本项目固定 `n_ref=5`, `epochs=30`, `steps_per_epoch=20`, `batch_size=48`, `seed=20260517`。
- **ADP-AHL-bridge-v2**: 后融合，不训练新模型。用 calib normal 对 ADP/AHL score 分别 robust normalize，再按固定 alpha 融合。alpha: S1-S2=0.70, S3-S4=0.60, S5-S7=0.35, S8=0.70；S0 无 bridge。
- **阈值口径**: calib 选阈值，test 只报告。S0-S2 用 best-F1@R>=0.90，S3-S6 用 best-F1@R>=0.85，S7-S8 用纯 best-F1。未使用 test oracle。

## 3. 一句话结论

GPU `gpu_tensor_uint8_aa_true` **可以基本替代 CPU/PIL 的 ADP-only 预处理**；但 **AHL-only 不是完美替代**，并且本轮 CPU 重跑与 v2 完全一致，说明 AHL 的 GPU 差异不能再主要归因为 CPU few-shot 重训随机性。对生产更相关的 bridge 口径，S1-S7 多数可接受，**S8 仍是主要风险点**。

## 7. 数据分析与决策建议

1. **ADP-only 可以视为 GPU 可替代 CPU。**  
   这是 training-free 口径，最大 AUPR 差仅 0.0104，F1 差主要来自少量阈值邻域样本预测翻转。结合此前 tensor p95 bit-exact 结论，GPU uint8 预处理对 ADP 主体语义基本保真。

2. **AHL-only 不能宣称完美替代。**  
   本轮 CPU/PIL 重跑与 v2 表完全一致，说明 CPU 流程是可复现的；因此 GPU AHL 在 S0/S5/S8 的大幅漂移不能主要解释为 CPU 重训随机性。AHL 对 feature/reference 采样和训练更敏感，GPU uint8 的微小特征差异会被 AHL 放大。

3. **Bridge 是实际更重要的生产口径。**  
   S1-S7 中 bridge 的最大 |dF1| 为 0.0265，整体属于可接受或可进一步业务确认的范围；S8 的 dF1=-0.0686、dRecall=-0.1000，是唯一明确不稳的 stage。

4. **当前最合理的落地判断:**  
   GPU uint8 可作为 ADP-only 主指标和 bridge 主线区 S1-S7 的默认候选；S8 不建议直接无条件替换 CPU/PIL，需要保留 fallback 或单独处理。AHL-only 结果应继续标注为“对 GPU 预处理敏感，不作为单独生产结论”。

## 8. 给理论/规划线程的精华

- CPU 完整重跑复现 v2: `CPU dF1 vs v2 = 0`，排除了“CPU few-shot 重训随机性很大”这个主要解释。
- ADP-only: GPU 替代基本成立，max |dF1|=0.0392，max |dAUPR|=0.0104。
- AHL-only: GPU 不等价，S0 AUPR 崩到 0.2133，S5 F1 -0.1587，S8 recall -0.2571。
- Bridge: S1-S7 基本稳，S8 仍退化；生产决策应把 S8 单列。
- 固定 CPU 阈值无法完全修复 S8，说明不是单一阈值策略问题。
